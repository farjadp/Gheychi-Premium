# راهنمای استفاده

این فایل برای استفاده روزمره از بات و پنل مدیریت است.

## 1. اجرای پروژه

### اجرای بات

```bash
cd /Users/farjad/Downloads/Work-Studio/gheychi-premium
source .venv/bin/activate
python bot.py
```

### اجرای پنل مدیریت

در یک ترمینال جدا:

```bash
cd /Users/farjad/Downloads/Work-Studio/gheychi-premium
source .venv/bin/activate
python admin_panel.py
```

## 2. استفاده از بات در تلگرام

1. بات را در تلگرام باز کن
2. `/start` را بزن
3. اگر آیدی عددی‌ات را می‌خواهی، روی دکمه `آیدی من` بزن
4. لینک ویدئو را بفرست
5. کیفیت مورد نظر یا گزینه `فقط صدا` را انتخاب کن
6. منتظر بمان تا فایل ارسال شود

دستورهای مفید:

- `/menu`: نمایش منوی سریع
- `/plans`: نمایش همه پلن‌ها
- `/myplan`: نمایش پلن فعال
- `/usage`: نمایش مصرف فعلی پلن
- `/mylogs`: نمایش آخرین لاگ‌های شخصی
- `/myid`: نمایش آیدی عددی تلگرام برای ارسال به ادمین
- `/support`: تماس با پشتیبانی

## 3. استفاده از پنل مدیریت

آدرس پنل:

```text
http://127.0.0.1:8080
```

اگر `ADMIN_PASSWORD` در `.env` مقدار داشته باشد، مرورگر از تو username/password می‌خواهد.

در پنل این بخش‌ها را داری:

- `حداکثر حجم فایل`: سقف فایل قابل ارسال برای بات
- `فعال بودن دانلود`: اگر خاموش شود، همه درخواست‌های جدید رد می‌شوند
- `پلتفرم‌های مجاز`: فقط پلتفرم‌های تیک‌خورده پذیرفته می‌شوند
- `اختصاص اشتراک`: ثبت یا تمدید پلن برای یک `Telegram User ID`
- `کاربران و اشتراک‌ها`: نمایش پلن ثبت‌شده، پلن فعال و مصرف فعلی
- `لاگ‌ها`: خطاها، شروع بات، دانلودهای موفق و خطاهای ارسال

## 4. سناریوهای رایج

### فروش و فعال‌سازی اشتراک

1. کاربر داخل بات دستور `/myid` را می‌زند
2. کاربر آن عدد را برای ادمین می‌فرستد
3. ادمین در پنل، `Telegram User ID` را وارد می‌کند
4. پلن و تعداد ماه را انتخاب می‌کند
5. اشتراک بلافاصله روی کاربر اعمال می‌شود

### بستن موقت بات

در پنل، `فعال بودن دانلود` را بردار و ذخیره کن.

### محدود کردن به Instagram و Twitter

در پنل فقط همین دو پلتفرم را نگه دار و بقیه را غیرفعال کن.

### افزایش سقف فایل

مقدار `حداکثر حجم فایل` را بالا ببر و ذخیره کن.

توجه: اگر Bot API محدودیت داشته باشد، بالا بردن این عدد به‌تنهایی کافی نیست.

### بررسی سهمیه یک کاربر

- از خود بات: `/usage`
- از پنل: بخش `کاربران و اشتراک‌ها`

## 5. مشاهده لاگ‌ها

دو روش داری:

- از داخل پنل مدیریت
- از API پنل

### دریافت تنظیمات

```bash
curl http://127.0.0.1:8080/api/settings
```

### دریافت لاگ‌ها

```bash
curl http://127.0.0.1:8080/api/logs
```

اگر پنل پسورد داشته باشد، باید با Basic Auth درخواست بفرستی.

### دریافت لیست کاربران و اشتراک‌ها

```bash
curl http://127.0.0.1:8080/api/users
```

## 6. خطاهای رایج

### `409 Conflict`

یعنی چند instance از بات هم‌زمان اجرا شده‌اند.

```bash
pkill -f "bot.py"
python bot.py
```

### `Button_data_invalid`

این خطا قبلاً برای لینک‌های بلند دیده می‌شد و در نسخه فعلی با token داخلی رفع شده است.

### `Requested format is not available`

این مورد برای YouTube با آپدیت `yt-dlp` رفع شده است. اگر دوباره برگشت، dependencyها را دوباره نصب کن:

```bash
source .venv/bin/activate
pip install -r requirements.txt
```

### `File name too long`

در نسخه فعلی فایل‌ها با نام کوتاه و یکتا ذخیره می‌شوند و این خطا رفع شده است.

### کاربر می‌گوید سهمیه‌اش تمام نشده ولی دانلود رد می‌شود

این مورد را از دو جا چک کن:

- `/usage` داخل بات
- جدول `کاربران و اشتراک‌ها` داخل پنل

توجه: بازه‌های `روز`, `هفته`, `ماه` در نسخه فعلی بر اساس UTC محاسبه می‌شوند.

## 7. فایل‌های مهم

- `.env`: تنظیمات واقعی پروژه
- `.env.example`: نمونه تنظیمات
- `data/settings.json`: تنظیمات runtime
- `data/activity.db`: دیتابیس لاگ‌ها
- `plans.py`: تعریف رسمی پلن‌ها و محدودیت‌ها
